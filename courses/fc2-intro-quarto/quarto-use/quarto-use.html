Basic Use

Overview

In this unit, you will learn the fundamental skills needed to create, edit, and render basic Quarto documents. You will explore the essential structure of a Quarto file and common workflows for generating outputs in different formats.

Goals

By the end of this unit, you will:

  • Understand the basic anatomy of a Quarto document.
  • Create and render documents to multiple output formats.
  • Utilize parameters and caching to improve efficiency and reproducibility.

Reading

Structure of a Quarto Document

The Quarto software operates on a file called a Quarto document (.qmd). A standard Quarto file (.qmd) is a text document that consists of:

  1. YAML Front Matter: Metadata and document configuration.
  2. Markdown Body: Narrative text written in Markdown.
  3. Executable Code Chunks: Embedded code blocks executed during rendering. This is optional, you can have a Quarto document without executable code chunks.

Example of a Minimal Quarto Document

Here is an example of a minimal Quarto document:

---
title: "Simple Quarto Example"
format: html
---

Here is a simple example demonstrating the integration of R code and narrative.

summary(mtcars)

Anatomy of a Quarto Document

A Quarto document typically consists of three main components:

  1. YAML Front Matter: Metadata and output configurations defined at the beginning of the file.
  2. Markdown Text: Narrative content formatted with Markdown syntax.
  3. Code Chunks: Blocks of executable code embedded within the narrative.

Example:

---
title: "Example Quarto Document"
format: html
---

This document integrates Markdown text with executable code.


::: {.cell}

```{.r .cell-code}
head(iris)
```

::: {.cell-output .cell-output-stdout}

Rendering Multiple Formats

Quarto allows the generation of multiple output formats from a single source file. You can specify multiple outputs in your YAML front matter:

---
title: "Multi-format Document"
format:
  html: default
  pdf: default
---

To override and render a specific format:

quarto render multi-format.qmd --to pdf
