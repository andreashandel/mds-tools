# Slide Deck Rubric

## Structure
- YAML header matches the required template
- Marker comment present after YAML: `<!-- AUTO-GENERATED SLIDES: edit freely; regenerate with care -->`
- Slides in order: Overview, Goals, Reading subsections, Summary
- One slide per `##` subsection inside `.unit-reading`

## Content
- Overview: 1?3 bullets
- Goals: bullet list from unit goals
- Reading slides: 3?6 concise bullets per slide
- Summary: 3?5 bullets
- No verbatim paragraph copy; bullets are summaries
- No invented factual claims
- Slide count target: 10?18 (6?10 if unit is short)

## Speaker Notes
- Every slide has a `::: {.notes}` block
- Notes are conversational and instructional
- 60?130 words per slide
- Notes expand on bullets rather than repeat them

## Media & Layout
- Preserve any existing images referenced in unit content
- Use code blocks only when necessary and keep them short
- Two-column layout used when it improves readability

## Quality Checks
- Deck renders without syntax errors
- No empty slides
- All required sections present
